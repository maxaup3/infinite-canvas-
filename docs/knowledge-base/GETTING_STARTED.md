## ⚡ 快速命令参考

当你在 Cursor 中工作时，这些命令会很有用：

```bash
# 启动开发服务器
npm run dev

# 检查代码错误
npx tsc --noEmit

# 查看文件修改
git diff src/components/Canvas.tsx

# 回退文件修改
git checkout src/components/Canvas.tsx

# 提交代码
git add .
git commit -m "修复：说明修改内容"

# 查看提交历史
git log --oneline
```

## 🤔 FAQ（最常见的问题）

### Q: 我需要学编程吗？
**A:** 不需要。这份知识库教你如何与 Claude Code 协作，Claude 负责写代码。你只需要：
- 清晰描述问题
- 反馈体验质量
- 在浏览器中测试

### Q: Claude Code 能做什么？
**A:** Claude Code 可以：
- ✅ 读取和理解代码
- ✅ 诊断问题原因
- ✅ 提出多个解决方案
- ✅ 编写和修改代码
- ✅ 解释技术概念
- ✅ 读取和分析 PDF 文档

但 Claude 不能：
- ❌ 在浏览器中自动测试代码
- ❌ 最终决定产品方向（那是你的工作）
- ❌ 理解你没有说出来的隐含需求

### Q: 如果修改出错怎么办？
**A:** 不用担心！使用 git 可以一键回退：
```bash
git checkout src/components/Canvas.tsx  # 回退单个文件
git revert <commit-hash>                # 回退某个提交
```

修改是可逆的，所以可以大胆尝试新想法。

### Q: 多久才能看到修改效果？
**A:** 通常很快：
1. 描述问题 (2-5 分钟)
2. Claude 诊断 (1-3 分钟)
3. Claude 编写代码 (5-15 分钟)
4. 你在浏览器验证 (1-3 分钟)

总计：**10-30 分钟** 一个小功能或 bug 修复

### Q: 需要一直看着 Claude 工作吗？
**A:** 不需要。你可以：
1. 描述问题
2. 告诉 Claude 你会回来验证
3. 做其他事情
4. 30 分钟后回来查看结果

Claude 会一直工作直到完成。
